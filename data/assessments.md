# Assessment instruments

Both assessments contain 10 open-ended questions in Portuguese, the language in
which the study was conducted. Each item was scored with a three-point rubric
(0 = incorrect, 1 = partially correct, 2 = fully correct) and final scores were
normalized to the [0, 1] interval, as described in the paper.

The immediate post-test is also included in `experiment_config.yaml` (survey
"Mostre o que você aprendeu!"). The delayed retention test, applied one week
later, rephrased each question while preserving conceptual equivalence; the
side-by-side table below shows the item-level correspondence.

| # | Pós-teste imediato | Teste de retenção (1 semana depois) |
|---|---|---|
| 1 | Explique com suas palavras o que é herança em orientação a objetos. | Descreva, com suas próprias palavras, o conceito de herança na programação orientada a objetos. |
| 2 | Qual é a principal vantagem do uso de herança em programação orientada a objetos? | Quais benefícios a herança pode trazer para o desenvolvimento de software orientado a objetos? |
| 3 | Dê um exemplo de um cenário em que herança poderia ser utilizada. Explique quais classes poderiam participar da hierarquia. | Imagine um sistema do mundo real em que a herança faria sentido. Descreva esse cenário e indique possíveis classes envolvidas. |
| 4 | Explique por que nem toda relação entre classes deve utilizar herança. | Em quais situações o uso de herança pode não ser a melhor escolha? Justifique sua resposta. |
| 5 | Explique o que é polimorfismo em orientação a objetos. | Como você explicaria o conceito de polimorfismo para alguém que está aprendendo orientação a objetos? |
| 6 | Como herança e polimorfismo estão relacionados? | De que forma a herança contribui para a implementação do polimorfismo? |
| 7 | O que é uma classe abstrata? | Qual é o papel de uma classe abstrata dentro de um sistema orientado a objetos? |
| 8 | Explique o que é um método abstrato e por que ele é utilizado. | Para que serve um método abstrato e em quais situações ele pode ser útil? |
| 9 | Qual é a diferença entre uma classe abstrata e uma classe comum? | Compare uma classe abstrata com uma classe concreta, destacando suas principais diferenças. |
| 10 | Uma classe abstrata pode existir sem métodos abstratos? Explique. | É obrigatório que uma classe abstrata possua métodos abstratos? Explique seu raciocínio. |
